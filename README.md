контрольные вопросы:
1. Что такое виртуальное окружение Python, и зачем оно используется?
Виртуальное окружение — изолированная среда для Python-проектов, позволяющая управлять зависимостями отдельно от системного Python. Используется для избежания конфликтов версий пакетов между проектами.

2. Какая команда используется для создания виртуального окружения?
python -m venv имя_окружения

3. Как активировать виртуальное окружение в Windows и Linux/macOS?
Windows: имя_окружения\Scripts\activate
Linux/macOS: source имя_окружения/bin/activate

4. Какая команда позволяет установить пакет в Python? Приведите пример.
pip install имя_пакета

5. Как вывести список всех установленных пакетов в виртуальном окружении?
pip list или pip freeze

6. Какая библиотека используется для выполнения HTTP-запросов в Python? Как её установить?
requests. Установка: pip install requests

7. Для чего используется библиотека BeautifulSoup4?
Для парсинга HTML/XML-документов, извлечения и навигации по данным.

8. Какой модуль используется для сохранения данных в файл? Приведите пример кода.
Встроенный модуль open() для текстовых файлов, json для JSON, csv для CSV.

9. Какой метод используется в библиотеке requests для выполнения GET-запроса? Приведите пример.
requests.get(url)

10. Какие типы объектов поддерживаются в BeautifulSoup для поиска элементов на странице (например, по тегам)?
По имени тега, классу CSS (class_), идентификатору (id), атрибутам, CSS-селекторам (select), тексту.

11. Что такое HTML-парсинг? Какие его основные задачи?
Извлечение структурированных данных из HTML-кода. Задачи: обход DOM-дерева, поиск элементов, извлечение текста/атрибутов.

12. Какие команды в Git позволяют создать новую ветку и переключиться на неё?
git branch имя_ветки (создать), git checkout имя_ветки (переключиться) или git checkout -b имя_ветки (создать и переключиться).

13. Что необходимо сделать перед выполнением коммита в Git?
Добавить файлы в индекс: git add файл или git add .

14. Как сохранить список установленных пакетов в файл requirements.txt? Приведите пример команды.
pip freeze > requirements.txt

15. Какие потенциальные ошибки могут возникнуть при парсинге сайта, и как их можно решить?
Ошибки: 404/403 (проверить URL/заголовки), изменение структуры сайта (регулярное обновление парсера), блокировка IP (использовать задержки, прокси), кодировка (указать правильную кодировку), JavaScript-контент (использовать Selenium/Playwright).
